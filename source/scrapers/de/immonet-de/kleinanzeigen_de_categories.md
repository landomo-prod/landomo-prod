## URL PATTERN STRUCTURE

### Basic Category Structure
```
/s-[category-slug]/c[category-number]
```

### With Location Filter
```
/s-[category-slug]/c[category-number]l[location-id]
```

### With Property Type Filter
```
/s-[category-slug]/c[category-number]+[category-slug].[filter-name]:[filter-value]
```

### Examples:
- Main category: `/s-immobilien/c195`
- Subcategory: `/s-wohnung-mieten/c203`
- With filter: `/s-wohnung-mieten/c203+wohnung_mieten.wohnungstyp_s:etagenwohnung`
- With location: `/s-wohnung-mieten/c203l3331` (Berlin)

## NOTES

- Category numbers (c-numbers) are persistent identifiers
- Property type filters use the format: `+[category].[attribute]:value`
- Geographic filtering uses location codes (l-codes)
- Most categories support filtering by transaction type (buy/rent)
- Some categories like "Umzug & Transport" are service-related, not property listings
- The platform distinguishes between "Angebote" (offers/listings) and "Gesuche" (wanted ads)
