BUG #1: GraphQL Parameter Names
================================

CHANGE LOCATIONS:
  • src/scrapers/listingsScraper.ts (5 changes)
  • src/types/realingoTypes.ts (1 change)

WHAT WAS WRONG:
  The GraphQL query was using 'limit' and 'offset' parameters, but the
  Realingo API requires 'first' and 'after' (cursor-based pagination).

WHAT WAS FIXED:
  
  GraphQL Query Variables:
    From: $limit: Int, $offset: Int
    To:   $first: Int, $after: String
  
  GraphQL Query Parameters:
    From: limit: $limit, offset: $offset
    To:   first: $first, after: $after
  
  Method Signatures:
    From: fetchOffers(variables, limit=100, offset=0)
    To:   fetchOffers(variables, first=100, after?)
  
  Type Definitions:
    From: limit?: number; offset?: number;
    To:   first?: number; after?: string;

---

BUG #2: Incorrect Enum Values
==============================

CHANGE LOCATIONS:
  • src/scrapers/listingsScraper.ts (2 changes)
  • src/types/realingoTypes.ts (4 changes)
  • src/transformers/realingoTransformer.ts (2 changes)

WHAT WAS WRONG:
  The GraphQL query was using 'SALE' and 'OTHER' enum values, but the
  Realingo API requires 'SELL' and 'OTHERS'.

WHAT WAS FIXED:
  
  Transaction Type Enum:
    From: 'SALE' | 'RENT'
    To:   'SELL' | 'RENT'
  
  Property Type Enum:
    From: '...' | 'OTHER'
    To:   '...' | 'OTHERS'
  
  All Occurrences Updated In:
    • Type definitions (RealingoOffer, RealingoSearchVariables)
    • Method signatures (scrapeSales, scrapeByPropertyType)
    • Transformer mappings (transaction_type, property_type)

BEFORE AND AFTER COMPARISON
================================================================================

GraphQL Query - BEFORE (Broken):
  query SearchOffer($purpose: OfferPurpose, $limit: Int, $offset: Int) {
    searchOffer(
      filter: { purpose: $purpose }
      limit: $limit
      offset: $offset
    ) { total, items { id } }
  }
  
  Variables: { "purpose": "SALE", "limit": 100, "offset": 0 }
  Result: ❌ API REJECTS - Wrong parameter names and enum value

GraphQL Query - AFTER (Fixed):
  query SearchOffer($purpose: OfferPurpose, $first: Int, $after: String) {
    searchOffer(
      filter: { purpose: $purpose }
      first: $first
      after: $after
    ) { total, items { id } }
  }
  
  Variables: { "purpose": "SELL", "first": 100, "after": "0" }
  Result: ✅ API ACCEPTS - Correct parameter names and enum value
